# 邪恶小女孩黄金矿工：跨设备排行榜设计稿

**日期：** 2026-03-30  
**目标：** 游戏完成后保存分数与游戏时长，并显示本局在历史全服中的排名（可跨设备、可分享给朋友）。

## 1. 范围与约束

- 现有游戏前端保留：`index.html + src/main-standalone.js`
- 后端方案：`Supabase`（前端直连）
- 榜单规则：全部玩家混排（不做“仅自己”筛选）
- 昵称规则：首次输入并记住，下次默认使用

## 2. 总体架构

前端新增排行榜模块，职责如下：

- 初始化 Supabase 客户端
- 结算后提交本局成绩
- 拉取排行榜并计算本局排名
- 驱动结算层展示排名与榜单

数据流：

1. 游戏启动 -> 读取本地昵称
2. 若无昵称 -> 弹窗输入并保存本地
3. 本局结束 -> 提交 `score + duration_seconds`
4. 提交成功后 -> 查询榜单并计算排名
5. 结算层显示：本局排名 + Top20

## 3. 数据模型

Supabase 表：`game_scores`

字段：

- `id`: `uuid`，主键，默认生成
- `player_name`: `text`，玩家昵称（1-12 字符）
- `score`: `int`，本局分数
- `duration_seconds`: `int`，本局时长（当前玩法为 60 秒）
- `captured_husband_count`: `int`，可选趣味统计
- `created_at`: `timestamptz`，默认 `now()`

排序规则（全服混排）：

1. `score DESC`
2. `duration_seconds ASC`
3. `created_at ASC`

## 4. 页面与交互设计

新增 UI 区块：

- 结算层新增“本局历史排名：#N / M”
- 结算层新增排行榜列表（Top 20）
- 设置“改昵称”入口（按钮或小链接）

昵称流程：

- 首次启动无昵称：弹输入框，昵称必填
- 输入合法后写入 `localStorage.playerName`
- 后续自动复用；用户可主动修改

结算流程：

1. 先显示本局得分与胜负状态
2. 异步提交本局成绩
3. 异步读取榜单
4. 更新排名与榜单区域

## 5. 错误与降级策略

- 提交失败：提示“成绩未上传，排行榜暂不可用”
- 拉榜失败：提示“上传成功，榜单加载失败”
- 网络慢：显示“排行榜加载中...”
- 不阻塞游戏重开：即使排行榜异常，也可继续下一局

## 6. 安全策略（MVP）

使用 Supabase RLS，允许匿名读取和插入 `game_scores`，限制范围为该表。
说明：MVP 阶段不做强防作弊。后续可升级为 Edge Function 进行分数合法性校验。

## 7. 验收标准

- 两台设备可看到同一份榜单
- 同分时，短用时排名更高
- 本局显示的 `#N` 与榜单排序一致
- 刷新页面后昵称仍保留
- 榜单异常不会影响继续玩游戏
