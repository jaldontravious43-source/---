# 邪恶小女孩黄金矿工：手机号验证码登录设计稿

**日期：** 2026-03-30  
**目标：** 引入真正的跨设备账号登录（手机号 + 验证码），未登录不可开局，登录后按账号持久化昵称与战绩。

## 1. 范围与目标

- 认证方式：Supabase Auth（Phone OTP）
- 访问门槛：必须登录后才能开始游戏
- 昵称策略：首次登录强制设置昵称，后续跨设备复用
- 排行榜策略：全服混排不变，结算页仅显示本局名次
- 战绩查询：主页面可查全服 Top15（含时间）

## 2. 登录与进入流程

1. 页面启动后弹出登录弹窗（手机号 + 验证码）
2. 未登录状态下禁用“开始游戏”
3. 用户输入手机号后点击“发送验证码”
4. 输入验证码并完成登录
5. 登录成功后：
   - 关闭登录弹窗
   - 启用“开始游戏”
   - 进入昵称检查流程（首次要求设置）
6. HUD 提供“退出登录”按钮，点击后回到登录态

## 3. 数据模型调整

在 `game_scores` 增加账号关联字段：

- `user_id uuid`（关联 `auth.users.id`）

同时新增账号资料表（用于昵称）：

- `player_profiles`
  - `id uuid primary key`（= `auth.users.id`）
  - `nickname text`（1-12）
  - `created_at timestamptz`
  - `updated_at timestamptz`

## 4. 账号与昵称规则

- 登录成功后先读取 `player_profiles.nickname`
- 若无昵称：
  - 弹出昵称设置框（必填）
  - 保存到 `player_profiles`
- 若有昵称：
  - 直接用于 HUD 与成绩提交
- 支持“改昵称”，更新 `player_profiles.nickname`

## 5. 成绩提交与查询规则

成绩提交 payload 变更为：

- `user_id`
- `player_name`（昵称快照）
- `score`
- `duration_seconds`
- `captured_husband_count`

排序规则保持：

1. `score DESC`
2. `duration_seconds ASC`
3. `created_at ASC`

展示规则：

- 结算层：只显示本局名次 `#N`
- 战绩查询：全服 Top15（昵称、分数、游戏时间 `YYYY-MM-DD HH:mm`）

## 6. UI 设计

新增/调整：

- 登录弹窗：
  - 手机号输入框
  - 验证码输入框
  - 发送验证码按钮
  - 登录按钮
- HUD：
  - 登录状态（手机号脱敏或昵称）
  - 退出登录按钮

按钮控制：

- 未登录：开始游戏、战绩查询禁用
- 已登录：恢复可用

## 7. 异常与降级

- 发送验证码失败：显示错误提示，允许重试
- 验证码无效/过期：提示重新获取
- 登录态丢失：自动回登录弹窗
- 昵称读取失败：强制进入昵称设置流程
- 排行榜接口失败：不阻塞重开，显示错误信息

## 8. 安全策略

- `game_scores` 写入要求 `auth.uid() = user_id`
- `player_profiles` 仅允许用户读写自己的行
- 匿名用户不可写分

## 9. 验收标准

- 未登录无法开始游戏
- 手机号 OTP 登录成功后可开局
- 同账号跨设备登录可看到同一昵称与历史战绩
- 结算页只显示本局名次
- 战绩查询可显示全服 Top15 与正确时间格式
